5. Topology
-----------
{{- if eq .Values.architecture.mode "federated" }}
  Federated mode uses Mosquitto bridge connections between {{ .Values.broker.replicaCount }} brokers.
  This is not a shared-state MQTT cluster; sessions and retained messages remain
  broker-local except where bridge topic rules propagate messages.
{{- else }}
  Standalone mode runs one broker with predictable local persistence.
{{- end }}

8. Validation Commands
----------------------
  Check broker pods and services:
    kubectl get pods,svc,pvc -n {{ $namespace }} -l app.kubernetes.io/instance={{ .Release.Name }} -o wide

  Wait for broker readiness:
    kubectl wait --for=condition=Ready pod -n {{ $namespace }} -l app.kubernetes.io/instance={{ .Release.Name }},app.kubernetes.io/component=broker --timeout=300s

  Inspect generated Mosquitto configuration:
    kubectl exec -n {{ $namespace }} statefulset/{{ include "mosquitto.statefulsetName" . }} -- cat /mosquitto/config/mosquitto.conf

  Follow broker logs:
    kubectl logs -n {{ $namespace }} statefulset/{{ include "mosquitto.statefulsetName" . }} -c mosquitto --tail=200
